2021年折腾记录

自疫情蔓延以来，在家维持赋闲的状态已经超过一年。

闲着也是闲着，人一闲下来就容易惹是生非，捣鼓各种过去从来不会捣鼓的东西。

开年以来，把买了好久也没有看过的《C Primer Plus》拿出来读，配合中文版的PDF对照着看，看到诘屈聱牙的地方就回英文纸质版上找原文比对。发现了翻译版中的许多错误，讶异出版社的校对工作竟如此粗心。

不过C语言还是挺有意思，越学越有乐趣。尤其是在Python宣布自3.9的版本不再能够运行于Windows 7之后，更加觉得还是得用一下C这样对平台依赖没有那么大的语言来写自己的代码。

我对Windows 10本来也没有什么不满，但是它并不能支持我最喜欢的拼音加加，严重影响我中文的录入速度与准确性。只好为了中文而特意准备了一台计算机专门运行Windows 7。

之前为了打游戏买的最便宜的巧克力键盘键位实在是奇葩，加上手感不良，于是就又加了50％的预算买了一个新的标准键位的薄膜键盘，录入效率与正确率飞升。

也不能老是学习，偶尔也看看B站上的番剧，玩玩Steam上好早以前低价买了又一直没碰过的游戏。

通关了《Clannad》和《Tomoyo After ～It's a Wonderful Life～》，后者Steam上只有英文版，但是B站上有Up主上传了中文的视频，所以开两个窗口，一边玩英文版，一边看B站的视频来阅读中文的剧情。

我倒也不是读不懂英文，实在是觉得日式Galgame的剧情的精髓在于文本，而同为东方国家，很多只有东方国家的人才能懂的心情，翻译成英文之后就丧失殆尽，根本无法有效地表达人物的情感。

当然后来又在3DM上找到了汉化版的《智代アフター ～It's a Wonderful Life～》，用虚拟机把它从3DM的壳里面解压出来，拷贝到计算机里面运行。入了正版玩盗版，也很无奈啊。

又接触了《Steins;Gate》，习惯了它独特的画风之后，也喜欢上了它的剧情。

而它的后续的许多衍生作品，Steam上要么没有引进，要么只有英文版。好在万能的B站上有一些神通广大的Up主上传了各个衍生作品的视频，可以让我得以当一个云玩家。

不得不说这两作都是制作精良的催泪神作，无愧于它们的豆瓣评分，十分令人感动。

还在一个神奇的西班牙语的Blogger网站上找到了Key社的全部OST的专辑下载.虽然下载解压缩之后，发现里面还夹着VeryCD分享计划的txt文档。真有一种考古出土文物的错觉。

学习之余，重新整理我的硬盘，把过去的数据重新分类整理，发现GoPro录制的视频太大，就萌生了把它们重新压制，减小体积，减轻硬盘存储压力的念头。

在这个过程中，学到了一些powershell和FFmpeg的知识，还了解了编码器的发展现状，与使用Nvidia和Intel的显卡相关硬解压的简单操作。

在调整参数的过程中，意外发现我的显卡色彩范围一直都是Limit RGB，了解了相关的知识之后，还意外发现了Dell的显示器会在Nvidia与Intel的显卡设置中被识别为数字电视，是导致被自动识别为Limit RGB的元凶。

经过一番学习，推测问题是出在显示器的EDID上，尽管我用的是Dell U2417h自带的而且N卡也好，I卡也好，都会好心的帮你自动识别显示器的类别，以确定是显示器还是电视。尝试修改EDID未果，我的显示器依然被识别为电视，心塞。

都2021年了，为什么N卡也好，I卡也好，就不能出一个简单的选项，让用户自己来选择显示设备的类型呢，我自己亲自动手连接的显示器难道我还不清楚它究竟是电脑显示器还是电视吗？

这些问题其实也有用户在厂家的论坛里反映，但是很多时候厂家都是置若罔闻，非不能也，乃不为也。

好在近年来不管是I卡还是N卡都给了大家一个曲线救国的方案，允许把色彩范围手动调整为Full RGB。保存重启之后，又试着把显示器调整为sRGB模式，发现显示器的色彩发生了很大的变化，需要一些时间来重新适应。

既然显示器色彩调整了，就顺手把亮度和对比度也校正了一下，感觉这么多年一直都是错误地使用计算机，有点白瞎了在设备上投资的金钱。

另一个烦心事是我的Lenovo Thinkcentre M900 Tiny，这台计算机就是上文所提到的为了拼音加加和中文录入而特意购置的。

自我买回来，就一直有各种各样的小问题。其中升级到BIOS的一个版本后会经常性遭遇蓝屏与死机。

回滚到FWKTACA版本之后稳定了一段时间。

去年Black Friday的时候，低价买了一块三星的860 EVO的1TB SSD来扩容我的计算机存储。

大概是因为计算机的设备总量增加了，对系统的电力提出了更高的挑战，而外置的电源适配器又是20V × 3.25A ＝ 65W的直流输出，从物理上决定了功率的上限。

近一个月以来，或许是Bilibili与Acfun之类的网站的视频压制格式出现了变化，也或许是Firefox内置的解码器出现了更新，已经出现至少四次在播放视频的过程中系统出现失去响应或者蓝屏后死机的故障。

通过调阅Event Viewer，发现死机前出现如下事件：

    Event ID 37
    Source: Microsoft-Windows-Kernel-Processor-Power
    Type: Warning
    Description:
    The speed of processorxin group y is being limited by system firmware. The processor has been in this reduced performance state forz seconds since the last report.

通过查阅微软的相关页面，并没有得到什么有用的信息，用hwmonitor_1.43来检测CPU的功耗，发现在开启了Turbo Mode的情况下，CPU瞬时功耗可以暴增到38W，大概是导致系统出现死机的罪魁祸首。

想来不是很能理解，毕竟Lenovo的整机按说也是做了出厂的测试的，不应该有这样的问题。

试着下载Intel Processor Diagnostic Tool 64bit，官网上的版本是4.1.5.37.W.MP，结果运行后第一秒就出现了IMC Fail：

    System Information

    CPU 1 - IMC - Fail.

    IPDT64 Failed
    --- IPDT64 - Revision: 4.1.5.37
    --- IPDT64 - End Time: 4/10/2021 10:55:12 AM

继续搜索，发现有人说他升级了主板BIOS就好了。我虽然有过曾经升级过BIOS导致各种随机死机，不得不又把BIOS降回去的不愉快的经验，但还是决定依法照做。

去Lenovo官网查找BIOS的升级，看到2021年又出了新版的BIOS，出于对厂家工程师办事的信任，就咬咬牙升了，升完怀着期待的心情又跑了一遍IPDT，结果测试还是fail。

又搜索，看到有人说这是因为最新的ICDT跟老CPU不兼容，用老版本的跑一遍就行，结果我找遍了Intel的官网，又找遍了整个互联网，根本找不到哪里有老版本的ICDT下载。

又看到有些人声称，在兼容模式中Windows 7来运行就可以通过测试，试了一下，结果ICDT拒绝启动，说该版本的操作系统不支持。

那就还是让实践检验真理吧，最后跑了一遍系统自带的Windows Memory Diagnostics Tool，重启之后做完测试，没有遇到错误，决定先从BIOS里面把SpeedStep和Turbo关了继续用一阵子，看看死机不死机再说。

这几天又迷上了VS Code，装上了PEP8之后，对Python的代码格式化真心好用啊，变得特别美观了。
